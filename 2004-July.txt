I figured out why we can't upload anything, all of our directories are
owned by root.  I sent in a support request.

I came across install-log (http://install-log.sourceforge.net/)
earlier today which is an old lfs tool.

Basically I was thinking if Spiffy used install there would be no need
to include a filelist inside of the package, and there would be little
need for complicated uninstallation.  This would make creating
packages much easier.

Here's how I was thinking it would work...

just before loading and running the installation command(s) Spiffy
would automatically run "touch /var/log/install-logs/.timestamp", it
would then proceed to install everything, and when the install
completed it would run "install-log package#version".  This creates a
file that contains a list of all the files that have been
added/modified/etc since "touch /var/log/install-logs/.timestamp" was
run.  The list is by default located in /var/log/install-logs/ however
we could move it to wherever.  The list now functions as an easy way
to uninstall packages.  Just like install-log site says now a simple
command like "xargs rm < /var/install-logs/package#version" will
uninstall the package.

I like it, it would make creating packages just that much easier. :)
So what do you think? Yay or ney?

The HTML should look something like:
<form method="POST" action="sendit.php">
 Name: <input type="text" name="name" /><br />
 Email: <input type="text" name="email" /><br />
 Please input a description of what you'd like to do for Spiffy and
your skills:<br />
 <textarea name="desc" rows="25" cols="10"></textarea>
</form>

The PHP part (sendit.php):
   <?php
     $error = 0; //If there is an error it gets set to one

/**************************************************************
** Be sure to se the variable $email_to below            **
**************************************************************/
     $email_to = "your email address"; // Set this to the email
address you want it sent to.

     //Make sure stuff exists
     if( $name == "" ) {
      print "Error: No name given.<br />";
      $error = 1;
     }
     if( $desc == "") {
      print "Error: No description given.<br />";
      $error = 1;
     }
     if( $email == "") {
      print "Error: No email address given.<br />";
      $error = 1;
     }
     if( $error != 0 ) {
      print "Script execution stopped due to errors.<br />";
      exit;
     }
     
     //Send mail
     $from = $email;
     $subject = "Forty-two Devloper: ".$name;
     $to = $email_to;
     $message = $desc;
     mail( $to, $subject, $message, $from )
      or $error = 1;
     if( $error != 0 ) {
      print "Error sending email. Please try again.<br />";
      exit;
     }
     
     //Tell the user about it
     print "An email has been sent to $email_to - below is a copy of
it. Thank you for joining the Spiffy team. You will be contacted
within a day.<br />";
     print "<br />";
     print "From: $email<br />";
     print "To: $email_to<br />";
     print "Subject: Forty-two Developer: $name<br />";
     print "<br />";
     print $desc."<br />";
   ?>

Make sure you include some HTML (like the template you're using for
the site) around sendit.php, too. Just don't put it inside the
<?php...?> tags. You may want to put the <?php...?> tags inside <p>
tags as it _will_ output stuff. :)

writes it to the HTML file. For example, <?php $paragraphit = "<p>";

source, it would say <p>Test</p>
