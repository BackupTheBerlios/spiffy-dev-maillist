I figured out why we can't upload anything, all of our directories are
owned by root.  I sent in a support request.

I came across install-log (http://install-log.sourceforge.net/)
earlier today which is an old lfs tool.

Basically I was thinking if Spiffy used install there would be no need
to include a filelist inside of the package, and there would be little
need for complicated uninstallation.  This would make creating
packages much easier.

Here's how I was thinking it would work...

just before loading and running the installation command(s) Spiffy
would automatically run "touch /var/log/install-logs/.timestamp", it
would then proceed to install everything, and when the install
completed it would run "install-log package#version".  This creates a
file that contains a list of all the files that have been
added/modified/etc since "touch /var/log/install-logs/.timestamp" was
run.  The list is by default located in /var/log/install-logs/ however
we could move it to wherever.  The list now functions as an easy way
to uninstall packages.  Just like install-log site says now a simple
command like "xargs rm < /var/install-logs/package#version" will
uninstall the package.

I like it, it would make creating packages just that much easier. :)
So what do you think? Yay or ney?
